Attendance System Using ESP32-CAM — System Flow

  1. Student appears in front of the ESP32-CAM.
  2. ESP32-CAM performs lightweight face detection on-device.
  3. ESP32 captures a JPEG image and sends it via HTTP POST to the FastAPI backend over Wi-Fi.
  4. FastAPI receives the image, re-runs accurate face detection, and crops the face.
  5. Backend generates face embeddings using MobileFaceNet or ArcFace.
  6. Backend compares embeddings with stored user embeddings in MongoDB Atlas.
  7. Backend identifies the person and records attendance in the database.
  8. Optional: Image is uploaded to Cloudinary and the URL is stored in the DB.
  9. A dashboard (React hosted on Vercel) displays attendance logs in real time.
Deployment note: ESP32 only does detection. All recognition happens on the backend for reliability.
